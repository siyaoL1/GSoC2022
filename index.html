Design challenge -- Enhancing the Camera operation

Part 1:

The issue of ‘useFront’ property stopped working was mentioned in issue #180, and the property was deprecated in PR #335. In the function capturePicture() at components/runtime/Camera.java, the ‘useFront’ property is checked for adding the intent ‘android.intent.extras.LENS_FACING_FRONT’. However, this intent is being removed in Android 5.1 (API level 22).

Solution of fix:


        import android.hardware.camera2.CameraCharacteristics;

        int frontCamera = CameraCharacteristics.LENS_FACING_FRONT;
        if (useFront) {
          if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP_MR1 && Build.VERSION.SDK_INT < Build.VERSION_CODES.O) {
            intent.putExtra("android.intent.extras.CAMERA_FACING", frontCamera);  // API 24
          } else if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            intent.putExtra("android.intent.extras.LENS_FACING_FRONT", frontCamera); // API 27
            intent.putExtra("android.intent.extra.USE_FRONT_CAMERA", true);
          } else {
            intent.putExtra("android.intent.extras.CAMERA_FACING", frontCamera);  // API 21
          }
         }

Part 2:

The existing Camera component uses Intent to open the native Camera App on the phone to take a picture, and we can’t alter what’s the behavior in the native Camera App. Instead if we want to control the behavior and support different feature of image capturing we can use the Camera API in the Android framework. Due to the fact that the underlying implementation will be different from the original Camera component, and not every user would need this automatic feature in camera, I will create a new Snapshot component.

This component will allow user to take a picture without user intervention, and since the picture is taken automatically, I will also disable the preview for it.

The steps to create the new Snapshot component is as follow.

  1. Creating an Icon
    • The snapshot component needs a 16x16 png icon placed in appengine/src/com/google/appinventor/images.
    • Add references to the icon inside Images.java at appengine/src/com/google/appinventor/client/Images.java.
    • Add the icon to bundledImages at appengine/src/com/google/appinventor/client/editor/simple/palette/SimpleComponentDescriptor.java.
  2. Declare and implement the Snapshot class
    • Create a Snapshot class that extends AndroidNonvisibleComponent at .appinventor/components/src/com/google/appinventor/components/runtime/
    • This file is the source code of the component Snapshot, and the Camera API will be used in this file to take picture with camera.
    • Due to the fact that Camera.takePicture() is only valid when preview is alive, we can use a dummy SurfaceView to avoid preview opening up. It can be achieved as follow:
